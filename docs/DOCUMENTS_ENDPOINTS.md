# Эндпоинты для создания накладных

## POST /documents/incoming-invoice

Создание приходной накладной.

### Формат запроса:

```json
{
  "dateIncoming": "28.12.2025",
  "comment": "Комментарий к документу",
  "supplier": "707a8ef8-60c0-f07e-018a-f452cbcd454b",
  "invoice": "INV-001",
  "items": [
    {
      "id": 3181,
      "amount": 10.0,
      "price": 100.0,
      "sum": 1000.0
    }
  ]
}
```

### Поля:

- `dateIncoming` (обязательное): Дата в формате `dd.mm.YYYY`
- `comment` (опциональное): Комментарий к документу
- `supplier` (опциональное): iiko_id поставщика. Если не указан, используется фиксированный поставщик
- `invoice` (опциональное): Номер счет-фактуры
- `items` (обязательное): Список позиций документа
  - `id` (обязательное): ID товара из таблицы `items` (нашего)
  - `amount` (обязательное): Количество товара
  - `price` (обязательное): Цена за единицу
  - `sum` (обязательное): Сумма позиции

### Автоматически устанавливаемые значения:

- `status`: всегда `"NEW"`
- `defaultStore`: всегда `"5849a5b1-1a73-40c3-a2dd-fd32f35325a2"`
- `conception`: всегда `"7e97ff39-9c68-40d7-9993-0a5dc53016e8"`
- `conceptionCode`: всегда `"13"`
- `supplier`: если не указан, используется `"707a8ef8-60c0-f07e-018a-f452cbcd454b"`
- `product`: берется из таблицы `items` по `id` -> `iiko_id`

### Ответ:

```json
{
  "success": true,
  "message": "Приходная накладная успешно создана",
  "iiko_id": null,
  "document_id": 123
}
```

## POST /documents/outgoing-invoice

Создание расходной накладной.

### Формат запроса:

```json
{
  "dateIncoming": "28.12.2025",
  "comment": "Комментарий к документу",
  "accountToCode": "001",
  "items": [
    {
      "id": 3181,
      "amount": 5.0,
      "price": 100.0,
      "sum": 500.0
    }
  ]
}
```

### Поля:

- `dateIncoming` (обязательное): Дата в формате `dd.mm.YYYY`
- `comment` (опциональное): Комментарий к документу
- `accountToCode` (опциональное): Код счета
- `items` (обязательное): Список позиций документа
  - `id` (обязательное): ID товара из таблицы `items` (нашего)
  - `amount` (обязательное): Количество товара
  - `price` (обязательное): Цена за единицу
  - `sum` (обязательное): Сумма позиции

### Автоматически устанавливаемые значения:

- `status`: всегда `"NEW"`
- `defaultStoreId`: всегда `"5849a5b1-1a73-40c3-a2dd-fd32f35325a2"`
- `conceptionId`: всегда `"7e97ff39-9c68-40d7-9993-0a5dc53016e8"`
- `conceptionCode`: всегда `"13"`
- `productId`: берется из таблицы `items` по `id` -> `iiko_id`

### Ответ:

```json
{
  "success": true,
  "message": "Расходная накладная успешно создана",
  "iiko_id": null,
  "document_id": 124
}
```

## Особенности

1. **Преобразование ID товаров**: Эндпоинты принимают `id` товаров из нашей таблицы `items`, затем автоматически преобразуют их в `iiko_id` для отправки в iiko API.

2. **Сохранение в БД**: Документы сохраняются в локальную БД (`warehouse_documents` и `warehouse_document_items`) и отправляются в iiko API.

3. **Формат даты**: Используется формат `dd.mm.YYYY` для обеих накладных (как в тестовом скрипте).

4. **Статичные значения**: Всегда используются фиксированные значения для store, conception и supplier (для приходных), что упрощает работу с API.
